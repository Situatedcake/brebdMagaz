# BrebdMagaz

Полноценный учебный интернет-магазин одежды на `Node.js + Express + Vanilla JS` с JSON-хранилищем, авторизацией по токену, корзиной, заказами, админ-панелью и локальной загрузкой изображений.

## Текущий функционал

- Каталог товаров с карточками, ценами, брендом, категорией и статусом продажи.
- Горячий поиск по каталогу (фильтрация при вводе текста).
- Процедурная подгрузка каталога при скролле (infinite scroll через sentinel).
- Карточки товаров и страница товара с каруселью изображений.
- Страница товара с крупным главным фото, галереей миниатюр и кнопкой `Назад`.
- Выбор размера работает как radio-группа; добавить в корзину нельзя, пока размер не выбран (если размеры есть).
- Авторизация и регистрация с кнопкой `Показать/Скрыть` пароль.
- Регистрация с автологином сразу после создания аккаунта.
- Профиль пользователя по центру страницы, редактирование имени и аватара.
- Предпросмотр аватара при выборе файла.
- Корзина:
  - для гостя хранится в `localStorage`
  - для авторизованного пользователя хранится в `db/users.json`
  - при входе гостевая корзина синхронизируется с серверной
- Оформление заказа через модальное окно выбора/ввода адреса доставки.
- После оформления заказ получает статус `in_transit` (`В пути`).
- В профиле отображаются заказы пользователя и доступна кнопка `Отменить заказ` для активных статусов.
- Админ-панель:
  - добавление товаров
  - редактирование товаров в той же форме (режим редактирования)
  - кнопка `Добавить товар` меняется на `Сохранить изменения`
  - кнопка отмены режима редактирования
  - переключение `Снять/Вернуть` по текущему статусу
  - удаление товара
  - загрузка фото только файлами (до 8)
  - удаление уже добавленных фото в режиме редактирования

## Технологии

- Frontend: `HTML`, `CSS`, `Vanilla JavaScript (ES Modules)`
- Backend: `Node.js`, `Express`
- Загрузка файлов: `multer`
- Хранение данных: JSON-файлы (`db/*.json`)
- Хэширование паролей: `sha256` (Node `crypto`)

## Роли и доступ

1. Гость
- Просмотр каталога и карточек товара.
- Горячий поиск и прокрутка с подгрузкой.
- Добавление в корзину (локально).

2. Пользователь
- Регистрация, вход, выход.
- Профиль и смена аватара.
- Серверная корзина и оформление заказов.
- Просмотр своих заказов и отмена активных.

3. Администратор (`isAdmin: true`)
- Доступ к `/admin.html`.
- Полное управление товарами (создание/редактирование/удаление/снятие с продажи).

## Авторизация

- Авторизация реализована через токен в заголовке `Authorization: Bearer <token>`.
- Токен и срок действия сессии хранятся в `db/users.json`:
  - `sessionToken`
  - `sessionExpiresAt`
- Время жизни сессии: 12 часов.

## Хранение изображений

- Аватары: `server/uploads/avatars`
- Фото товаров: `server/uploads/products`
- Статика отдается по префиксу `/uploads/...`
- Ограничения загрузки:
  - аватар до 2 MB
  - фото товара до 5 MB за файл
  - до 8 файлов на товар
- Перед отправкой изображения сжимаются на клиенте (`canvas` + JPEG quality).

## Основные API

### Auth
- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `POST /api/auth/logout`

### Users
- `PUT /api/users/profile`

### Products
- `GET /api/products?q=&limit=&offset=`
- `GET /api/products/:id`

### Cart
- `GET /api/cart`
- `POST /api/cart`
- `PUT /api/cart`
- `DELETE /api/cart/:id?size=`

### Orders
- `GET /api/orders/my`
- `POST /api/orders`
- `PATCH /api/orders/:id/cancel`

### Admin
- `GET /api/admin/products`
- `POST /api/admin/products`
- `PUT /api/admin/products/:id`
- `PATCH /api/admin/products/:id/toggle-sale`
- `DELETE /api/admin/products/:id`

### Upload
- `POST /api/upload/avatar` (form-data: `avatar`)
- `POST /api/upload/product-images` (form-data: `images[]`, max 8)


2. Запустить проект:

```bash

```

## Ограничения текущей реализации

- Вместо БД используется файловое JSON-хранилище.
- Нет транзакций и блокировок записи при параллельных изменениях.
- Нет автоматической очистки файлов из `server/uploads`, если изображения перестали использоваться.
- Пароли хэшируются `sha256` без `salt` (для продакшна требуется более безопасная схема, например `bcrypt`).
- Нет автоматических тестов (`unit/e2e`) и CI-пайплайна.
