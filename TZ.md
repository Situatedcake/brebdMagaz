## Суть проекта

Это интернет-магазин брендовой одежды, где пользователь может:

- просматривать каталог товаров
- искать товары
- открывать отдельную страницу товара
- добавлять товары в корзину
- удалять товары из корзины
- оформлять заказ из корзины
- регистрироваться и авторизовываться
- редактировать профиль

## Сессия и кэш

Обязательно реализуй следующую логику:

### Активная сессия

- активная сессия пользователя должна сохраняться в кэш
- если пользователь обновляет страницу, сессия не должна теряться сразу
- можно использовать localStorage/sessionStorage на клиенте и серверную проверку токена/сессии в упрощенном виде

### Корзина гостя

- если пользователь не авторизован, его корзина должна сохраняться в кэш
- после авторизации корзина гостя должна переноситься в корзину пользователя в БД или в его серверное состояние

То есть нужна логика:

1. гость добавил товары в корзину
2. корзина лежит в localStorage
3. пользователь зарегистрировался или вошел
4. товары из localStorage переносятся в данные пользователя
5. localStorage-корзина синхронизируется

## Структура сайта

Сделай минимум 3 страницы:

### 1. Главная страница — `index.html`

На странице должно быть:

- шапка сайта

- кнопки перехода по страницам
- если пользователь **не авторизован**:

  - кнопка "Зарегистрироваться"

- если пользователь **авторизован**:
  - иконка или фото пользователя
  - кнопка перехода в профиль
  - кнопка выхода

Также на главной странице:

- строка поиска по товарам
- каталог товаров
- товары должны подгружаться **по мере прогрузки страницы**

  - lazy render
  - кнопку "Показать еще"
  - либо бесконечную подгрузку

- каждый товар должен отображаться карточкой:
  - фото
  - название
  - бренд
  - цена
  - кнопка перехода в товар
  - кнопка добавить в корзину

### 2. Страница товара

Для каждого товара должна быть отдельная страница логически, **но в проекте физически должна лежать только одна HTML-страница-шаблон**, например:

На странице товара должно быть:

- большое изображение товара
- название
- бренд
- цена
- описание
- размер или доступные размеры
- статус: в продаже / снят с продажи
- кнопка добавления в корзину
- кнопка перехода в корзину

---

### 3. Страница корзины

На странице корзины должно быть:

- список добавленных товаров
- фото товара
- название
- цена
- количество
- кнопки `+` и `-` для количества
- кнопка удаления товара
- итоговая сумма
- кнопка оформления заказа

Важно:

- если пользователь не авторизован, корзина должна отображаться
- но при попытке оформить заказ нужно показать сообщение:
  - что для оформления заказа нужна авторизация

## Важные правила реализации

1. Пиши код **как junior разработчик**:
   - без переусложнения
   - без TypeScript
   - без React/Vue/Angular
   - без ORM
   - без MongoDB/PostgreSQL/MySQL
   - без лишних абстракций

2. Код должен быть:
   - понятным
   - читаемым
   - разделенным по файлам
   - с комментариями в сложных местах

3. **Обязательно добавляй комментарии** там, где логика может быть непонятна начинающему разработчику.

4. Не делай всё в одном файле.

5. Добавь начальные тестовые данные:
   - несколько пользователей
   - минимум 12–20 товаров
   - один админ

6. Пароль можно хранить упрощенно для учебного проекта, но лучше показать базовую безопасную структуру. Если используешь хеширование — отлично. Если нет, явно укажи в README, что это учебное упрощение.

7. Покажи обработку ошибок:
   - неверный логин/пароль
   - пользователь уже существует
   - товар не найден
   - недостаточно прав
